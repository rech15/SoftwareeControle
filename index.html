Softwares de Versionamentos

São sistemas que tem por objetivo realizar o controle de versões de um código ou documento. É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipes de desenvolvedores.

Software de Controle de Versões

Para realizar esse controle de versões, é através de um software que vai criar um Repositório Local, e com isso, vai criar apartir do seu arquivo um histórico de versões. Além do Repositório Local, pode ser usado um Repositório Remoto que é usado principalmente quando o trabalho é em equipe, mas também tendo ultilidade individualmente, os arquivos e projetos serão salvos na nuvem. Funciona da seguinte maneira, o usuário terá um servidor específico para trabalhar com seu Repositório Remoto juntamente o gerenciando. Com isso o Software de Controle de Versões vai gerenciar tanto o histórico de Repositório Local quanto o Remoto fazendo sempre essa sincronização.

Tipos de Software de Controle de Versão

  • Centralizado
  • Distribuído
Centralizado
Possui um repositório central em um servidor com controles de versão. Depende do servidor. Software Subversion.
Distribuído
Possui um repositório local com controles de versões. Atualizações do repositório central. Independente do servidor. Software Mercurial, Git

Git

O Git é um software livre de versionamento que surgiu para o desenvolvimento do Karnel do Linux, foi projetado e desnvolvido por Linus Torvalds em 2005. Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.

"I'm an egotistical bastard so I name all my projects after myself. First Linux now Git." (Linus Torvalds).

GitHub

É uma plataforma de hospedagem de código fonte e arquivos com controle de versão. Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por US$ 7,5 bilhões. Hoje o GitHub hospeda cerca de 100 milhões de projetos com 56 milhões de usuários.

Comandos GIT

  • commit
  • push
  • fetch
  • pull
commit
Torna permanente um conjunto de alterações em um repositório.
push
Envia alterações para o servidor no repositório remoto.
fetch
Verifica se o seu repositório local está em sincronia com o remoto.
pull
Sincroniza o repositório local com o remoto atualizando as versões.
